Updating Events

Updating events requires the updating of the /js/events.js file. To update events:

  1. Open the /js/events.js file in an editor
  2. Copy and paste an exsisting event where it occurs on the timeline of events
  3. Update the event info
  4. When done editing, save the file
  5. Save the events image in to the /img/event-images folder and copy and paste the file name you updated in the events.js to rename it
  6. In the command prompt, navigate to /python/ and run "python build.py". This does a number of things including:
    • Updating the /js/events.json/ file with the new events from the events.js file. The home page, calendar page, and past events page all use this to build their event timelines (I think, main.js should be looked at)
    • Creates a minimized version of the events.js file (events-min.js)
    • Takes the urls from the JSON file, including the ones that you added while editing and builds new populated HTML files for all events and places them in the special events folder.
  7. In a FTP, login to the hosting
  8. Copy event.js, events.json, and events-min.js into the /public_html/js/ folder, replacing the old files
  9. Copy the event image file into the /img/event-images/ folder
  10. Copy the speciffic event HTML pages that we created (NOTE: you dont need to upload all the html files)
  11. Close the FTP
  12. Upload the new files to GitHub. In the command prompt:
    1. git add --a
    2. git commit -m "What you did in this version"
    3. git push
  13. Check the site.
